# Исходная формулировка задания (ORM, Hibernate)

Ниже зафиксирована типовая постановка учебной работы по теме «ORM, Hibernate» для курса Java. При выдаче диспетчером другого текста или вложений замените этот файл фактическими материалами и сохраните их в каталоге `Docs/`.

## Цель

Освоить базовые приёмы объектно-реляционного отображения на примере Hibernate: настройка `SessionFactory`, маппинг сущности на таблицу, выполнение операций создания, чтения, обновления и удаления данных.

## Требования

1. Настроить проект (Maven) с зависимостями Hibernate ORM и драйвером СУБД (в решении используется встраиваемая H2).
2. Описать хотя бы одну сущность с аннотациями JPA (`@Entity`, `@Table`, `@Id`, `@GeneratedValue`, поля с `@Column`).
3. Реализовать слой доступа к данным (DAO или репозиторий) с использованием `Session` и транзакций.
4. В точке входа продемонстрировать сценарий: сохранение записей, выборка списка, обновление, удаление.
5. Обеспечить осмысленное логирование этапов работы приложения.

## Критерии приёмки

- Проект собирается командой `mvn -q -DskipTests package` (или без `-DskipTests`, если добавлены тесты).
- Приложение запускается и завершается без необработанных исключений.
- В консоли и/или логах видны операции с БД и итог демонстрационного сценария.

## Примечание

Если к заданию прилагались файлы из отдельного репозитория (шаблоны, XML, скриншоты), их копии следует хранить в `Docs/` рядом с этим файлом.
